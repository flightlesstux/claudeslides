# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**claudeslides** is an open-source gallery at [claudeslides.com](https://claudeslides.com) (GitHub Pages) where anyone — kids, seniors, students, hobbyists — can submit an HTML slide/showcase page demonstrating what they built or learned with Claude. The philosophy: in the near future, everyone will know a little coding, and this repo proves it.

## Repository Structure

```
claudeslides/
├── index.html              # Main gallery (GitHub Pages root — DO NOT delete)
├── submissions.json        # Auto-generated by CI; lists all author entries
├── CONTRIBUTING.md         # Contributor guide
├── author/
│   └── {slug}/
│       └── index.html      # Each contributor's self-contained slide page
└── .github/
    ├── workflows/
    │   ├── pr-moderation.yml    # Content safety check on every PR
    │   └── update-gallery.yml   # Regenerates submissions.json on merge to main
    ├── scripts/
    │   ├── moderate_content.py  # Content safety scanner
    │   └── generate_submissions.py  # Builds submissions.json from author/ dirs
    └── PULL_REQUEST_TEMPLATE.md
```

## Contribution Pattern

Each contributor:
1. Creates a new folder `author/{their-chosen-slug}/`
2. Adds a single `index.html` — a self-contained page (all CSS/JS inline or CDN)
3. Opens a PR — CI moderates content then maintainer reviews

The `{slug}` becomes part of the public URL: `claudeslides.com/author/{slug}/`

## CI Pipelines

### PR Moderation (`pr-moderation.yml`)
- Triggered on every PR that touches `author/`
- Runs `moderate_content.py` which extracts visible text from the HTML and checks for harmful content (hate speech, sexism, CSAM references, harassment, etc.)
- Posts result as a PR comment and sets a status check
- Uses pattern matching + optional Ollama (if self-hosted runner has it)

### Gallery Update (`update-gallery.yml`)
- Triggered on push to `main` (after merge)
- Runs `generate_submissions.py` which scans `author/*/index.html`, extracts `<title>` and metadata, writes `submissions.json`
- Commits the updated `submissions.json` back to main

## Key Rules for AI Assistance

- **`index.html` at root = the gallery.** Never redirect it or break the gallery fetch of `submissions.json`.
- **Each `author/*/index.html` is standalone.** No shared framework, no build step. Pure HTML.
- **`submissions.json` is generated — never edit manually.** Always regenerate via the script.
- **Content moderation is non-negotiable.** Do not weaken or bypass `moderate_content.py`.
- **GitHub Pages serves from `main` branch root.** No `gh-pages` branch needed.

## Sitemap Maintenance (sitemap.xml)

**Always keep `sitemap.xml` in sync** — it is hand-maintained (not auto-generated):

- **Presentation added:** add a `<url>` entry with `https://claudeslides.com/author/{slug}/{project}/`, `<lastmod>` set to today, `<changefreq>yearly</changefreq>`, `<priority>0.6</priority>`.
- **Presentation removed:** delete its `<url>` block.
- **Author slug renamed:** update every matching `<loc>` entry.
- **Page deleted (e.g. `/contribute/` was merged into `index.html`):** remove its `<url>` block immediately.
- Never add URLs that don't resolve to real pages on the live site.

## Open-Source Hygiene Rules

- **Author slug = the folder name under `author/`.** It forms the public URL (`claudeslides.com/author/{slug}/`). Never rename a slug without also updating: canonical/og URLs inside that author's HTML files, `submissions.json` paths, and `sitemap.xml`.
- **GitHub profile/repo links stay as the contributor's actual GitHub username.** Do not conflate the author slug with the GitHub username — they can differ.
- **`/contribute/` no longer exists as a separate page** — it was merged into `index.html`. Do not recreate it or link to it.
- **Do not touch `submissions.json` manually.** It is fully auto-generated by `.github/scripts/generate_submissions.py` on every push to `main`.
- **Never weaken CI checks.** PR moderation and gallery-update workflows must remain intact and untampered.
- **No build step, no dependencies.** The repo must remain forkable and deployable with zero setup — pure HTML, inline CSS/JS, or CDN only.

## Metadata Convention in Author Pages

Author pages should include these meta tags for the gallery to extract richer data:

```html
<meta name="author" content="Your Name">
<meta name="description" content="One-line summary of what you built">
<meta name="og:image" content="https://...">  <!-- optional preview image -->
```

The `generate_submissions.py` script reads `<title>`, `meta[name=author]`, and `meta[name=description]`.
