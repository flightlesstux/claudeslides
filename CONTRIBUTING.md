# Contributing to claudeslides

Welcome! This project exists because **everyone deserves to share what they create with Claude** — whether you're a 10-year-old who built their first game, a grandmother who organized her recipes, or a developer who shipped something wild. If Claude helped you make it, it belongs here.

## What to Submit

A single HTML file that showcases something you did with Claude. It could be:

- A project walkthrough or case study
- A creative experiment (art, music, writing, games)
- A tutorial you built
- A tool or automation that changed your workflow
- Anything that made you proud

The page can look however you want — slides, a scrolling story, a demo, a gallery.

## How to Submit

### Step 1 — Fork this repository

Click **Fork** on GitHub. You now have your own copy.

### Step 2 — Create your folder

Inside the `author/` directory, create a new folder with a URL-friendly slug (lowercase, hyphens, no spaces):

```
author/your-name/
```

For example: `author/maria-from-brazil/` or `author/5th-grade-robotics-club/`

### Step 3 — Add your `index.html`

Create `author/your-name/index.html`. This is your canvas — one self-contained HTML file. All CSS and JavaScript must be inline or loaded from a public CDN. No npm, no build steps, no external files in the repo.

**Required meta tags** (copy this into your `<head>`):

```html
<meta name="author" content="Your Name">
<meta name="description" content="One sentence about what you built with Claude">
```

### Step 4 — Open a Pull Request

Push your branch and open a PR against `main`. Use the PR template that appears automatically.

An automated content check will run within a minute. If it passes, a maintainer will review and merge — usually within a few days.

## Content Guidelines

We welcome everyone. To keep this space safe:

**Not allowed:**
- Hateful, discriminatory, or dehumanizing content (racism, sexism, homophobia, etc.)
- Any content that sexualizes or harms minors
- Harassment or personal attacks
- Spam, scams, or malicious code
- Disinformation presented as fact

**Automated checks catch:**
- Slur lists and hate speech patterns
- CSAM-related language
- Injection attacks or malicious scripts in HTML

PRs that fail the automated check will receive a comment explaining why and will not be merged until resolved.

## Technical Rules

| Rule | Why |
|------|-----|
| One folder per submission: `author/{slug}/index.html` | Keeps the gallery organized |
| All assets inline or CDN | No binary files or asset blobs in the repo |
| No changes to `index.html` at root | That's the gallery — maintainers manage it |
| No changes to `submissions.json` | Auto-generated by CI |
| No changes to `.github/` | Maintainers only |

## Updating Your Submission

Open another PR with changes only inside your `author/{slug}/` folder. Same review process.

## Questions?

Open a [GitHub Discussion](../../discussions) or an Issue. We're friendly.
