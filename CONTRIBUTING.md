## Submission Specification

A valid submission is exactly **one file**:

```
author/{author-slug}/{slide-slug}/index.html
```

### Slug Rules
- Lowercase letters, digits, and hyphens only
- No spaces, underscores, or special characters
- `{author-slug}` — your name/handle, consistent across all your submissions (e.g. `maria-from-brazil`)
- `{slide-slug}` — short name for this specific project (e.g. `my-recipe-app`)
- Examples: `author/maria-from-brazil/recipe-app/`, `author/tokyo-dev/rust-cli-tool/`

### Required `<head>` Meta Tags

```html
<meta name="author" content="Author Name">
<meta name="description" content="One sentence about what was built with Claude">
```

> **CI auto-generates and injects all image and SEO tags** (via `generate_covers.py` + `inject_seo.py`).
> Contributors must NOT provide `og:image`, `twitter:image`, or any cover image — they are always
> overwritten by the pipeline with the CI-generated branded `cover.jpg`.
>
> You only need to provide `author` and `description`. Everything else is handled automatically.

## File Constraints

| Constraint | Value |
|-----------|-------|
| Files allowed per PR | 1 (`author/{slug}/index.html`) |
| CSS/JS | Inline or public CDN only |
| Binary assets | Not allowed in repo |
| External files in `author/{slug}/` | Not allowed |

### Files that MUST NOT be modified

- `index.html` (root) — gallery page, maintainer-managed
- `submissions.json` — auto-generated by CI
- `.github/` — CI workflows, maintainer-managed
- `CLAUDE.md`, `CONTRIBUTING.md`, `README.md` — maintainer-managed

## CI Pipeline Behavior

### On PR open or update (`pr-moderation.yml`)

1. Triggers when `author/**` is touched
2. Runs `moderate_content.py` — extracts visible text from HTML
3. Checks for: hate speech, slurs, CSAM references, harassment, malicious scripts
4. Posts result as a PR comment
5. Sets status check: `Content Safety Check` (required to merge)

### On merge to `main` (`update-gallery.yml`)

1. Runs `generate_covers.py` — always generates `cover.jpg` (1200×630) for every submission (overwrites any contributor-provided image)
2. Runs `inject_seo.py` — injects missing SEO/OG/Twitter meta tags into each `index.html` (non-destructive)
3. Runs `generate_submissions.py` — extracts metadata, writes `submissions.json`
4. Runs `generate_sitemap.py` — regenerates `sitemap.xml` with all current submissions
5. Commits all changes back to `main`

## Content Policy — Automated Rejection Criteria

Submissions are automatically rejected if they contain:

- Hate speech, slurs, or dehumanizing language (racism, sexism, homophobia, etc.)
- Any sexualization or harm targeting minors
- Harassment, doxxing, or personal attacks
- Malicious code, XSS payloads, or injection attacks
- Spam, phishing, or disinformation

## PR Requirements

- Touch only `author/{slug}/` — one folder, one `index.html`
- PR title: short description of what was built
- Use the auto-populated PR template
- Do not modify any shared or root-level files
