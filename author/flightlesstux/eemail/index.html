Key Architecture Decision

"Keep emails on the server.
Never download fully."

// OLD: download everything on sync
for await (const msg of client.fetch(range, { source: true })) {
  // 10MB email? Downloaded. Every time. Painful.
}

// NEW: metadata only during sync (Claude's idea)
for await (const msg of client.fetch(range, {
  uid: true, flags: true, envelope: true,
  headers: true, bodyStructure: true, size: true
})) { // body fetched on-demand when user opens email ✓ }
Headers only sync Body fetched on open Valkey 300s body cache Attachments parsed + stored once
